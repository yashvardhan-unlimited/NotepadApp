{# LEARNING NOTE: One form, two routes ===================================== This single editor.html is used for BOTH: 1. Creating a new note (action="/note/new") 2. Editing an existing note (action="/note/edit/{{ note._id }}") We use Jinja2 to set the "action" attribute dynamically. If "note" exists → post to the edit URL with the note's ID. If "note" is None → post to the create URL. FastAPI has separate @app.post() handlers for each URL. #}
{# LEARNING NOTE: Pre-filling the form ===================================== When editing an existing note, we want the form to already contain the note's current data. "value=" sets the initial value of the input field. {% if note %}{{ note.title }}{% endif %} → If note exists, put note.title as the pre-filled value. → If it's a new note, leave value empty (no value attribute). #}
{# LEARNING NOTE: Pre-filling a textarea ====================================== Textareas don't use the "value" attribute like regular inputs. Instead, you put the text BETWEEN the opening and closing tags: So we do: {% if note %}{{ note.content }}{% endif %} inside the textarea tags to pre-fill it when editing. Note: There's no space/newline before the content to avoid adding extra whitespace at the beginning. #}
